# MLP_and_CNN_on_MNIST_CIFAR_ImageClassifacation

# Семинар по PyTorch: классификация изображений

## Описание
Блокнот посвящён знакомству с базовым пайплайном обучения моделей в PyTorch на задаче классификации изображений.  
На примере **MNIST** и **CIFAR-10** разбираются загрузка данных, построение моделей, обучение, отладка и улучшение качества.

## Что есть в блокноте
- работа с `Dataset` и `DataLoader`;
- преобразование изображений в тензоры;
- обучение полносвязной сети (**MLP**);
- сравнение поведения модели на **MNIST** и **CIFAR-10**;
- проверка сходимости через переобучение на одном батче;
- разработка на малой подвыборке;
- аугментации данных;
- переход от MLP к **CNN**;
- добавление **Batch Normalization**;
- настройка **learning rate scheduler**;
- сравнение оптимизаторов **SGD** и **Adam**;
- использование **transfer learning** и **fine-tuning** на базе `ResNet`.

## Датасеты
### MNIST
Набор рукописных цифр:
- 10 классов;
- изображения `1x28x28`.

### CIFAR-10
Набор цветных изображений:
- 10 классов;
- изображения `3x32x32`.

## Архитектуры
### 1. MLP
Базовая полносвязная сеть:
- `Linear`;
- `ReLU`;
- выходной линейный слой на 10 классов.

Используется как стартовая модель для проверки пайплайна.

### 2. CNN
Сверточная сеть в стиле LeNet:
- несколько `Conv2d`;
- `ReLU`;
- `MaxPool2d`;
- классификационная голова из линейных слоёв.

### 3. CNN с BatchNorm
Расширение CNN с добавлением `BatchNorm2d` для стабилизации обучения.

### 4. Transfer Learning
Используются предобученные модели:
- `ResNet18`;
- `ResNet50`.

Рассматриваются варианты:
- обучение только новой классификационной головы;
- частичная разморозка верхних слоёв;
- обучение всей сети целиком.

## Обучение
В блокноте реализован общий цикл обучения с:
- переносом модели и данных на `CPU/GPU`;
- вычислением `CrossEntropyLoss`;
- подсчётом accuracy на train и validation;
- визуализацией loss;
- поддержкой scheduler.

Используемые оптимизаторы:
- `SGD` с momentum;
- `Adam`.

## Аугментации
Для борьбы с переобучением используются:
- `RandomAffine`;
- масштабирование;
- повороты;
- сдвиги;
- `RandomHorizontalFlip`.

## Основные выводы
- MLP подходит для демонстрации базового пайплайна, но плохо масштабируется на более сложные изображения.
- На **CIFAR-10** полносвязные сети быстро упираются в ограничение по качеству.
- **CNN** работает лучше, чем MLP, за счёт извлечения локальных признаков.
- **BatchNorm** делает обучение стабильнее и помогает улучшить обобщающую способность.
- Аугментации снижают переобучение на малых выборках.
- Для сложных данных наилучший эффект даёт **transfer learning** с предобученными `ResNet`.

## Итог
Блокнот показывает типичный путь улучшения модели в задачах компьютерного зрения:
1. проверить базовый пайплайн на простой модели;
2. убедиться в сходимости на одном батче и малой подвыборке;
3. добавить аугментации;
4. перейти к CNN;
5. стабилизировать обучение через BatchNorm и scheduler;
6. использовать transfer learning для достижения лучшего качества.
